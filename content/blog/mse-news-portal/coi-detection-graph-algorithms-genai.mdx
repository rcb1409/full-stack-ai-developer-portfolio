Academic conflict-of-interest review is tedious and error-prone. Before this system existed, an MSE department administrator would manually search publications databases to find co-authorship relationships between grant reviewers and applicants — a process that took 2 hours per review cycle across 27 faculty members.

The automated system does the same work in **0.5 seconds** with 98% precision.

## What COI means in this context

A conflict of interest exists when a reviewer has a professional relationship with a grant applicant that might bias their evaluation. The most common forms in academia:

- **Direct co-authorship**: the reviewer and applicant published a paper together
- **Indirect co-authorship**: they share a co-author (one hop away in the collaboration graph)
- **Institutional affiliation**: same department, same lab group

The existing rule at NCSU: flag any reviewer who co-authored with the applicant within the last 5 years, or who has a collaborator in common within 3 years.

## Data model: a collaboration graph

The first step was modeling the data as a graph. I imported 3,605 publications and 23,048 author records into a graph structure:

- **Nodes**: researchers (identified by OpenAI-generated canonical IDs + ORCID cross-referencing)
- **Edges**: co-authorship relationships, weighted by recency (publications from 2024 weighted higher than 2019)

```python
import networkx as nx
from collections import defaultdict

def build_collaboration_graph(publications: list[dict]) -> nx.Graph:
    G = nx.Graph()
    
    for pub in publications:
        authors = pub['authors']  # List of canonical author IDs
        year = pub['year']
        
        # Add all authors as nodes
        for author_id in authors:
            G.add_node(author_id)
        
        # Add edges between all author pairs (co-authorship)
        for i, author_a in enumerate(authors):
            for author_b in authors[i+1:]:
                if G.has_edge(author_a, author_b):
                    # Track multiple co-authored papers and recency
                    G[author_a][author_b]['papers'].append(pub['id'])
                    G[author_a][author_b]['latest_year'] = max(
                        G[author_a][author_b]['latest_year'], year
                    )
                else:
                    G.add_edge(author_a, author_b, papers=[pub['id']], latest_year=year)
    
    return G
```

After processing all 3,605 publications, the graph had **6,973 unique researcher nodes** and **18,420 edges**.

## COI detection: BFS with a 5-year cutoff

Given a reviewer and an applicant, I run a BFS limited to 2 hops and 5 years:

```python
from datetime import datetime

CURRENT_YEAR = datetime.now().year
COI_HORIZON_YEARS = 5
COI_INDIRECT_HORIZON = 3

def check_coi(
    graph: nx.Graph,
    reviewer_id: str,
    applicant_id: str
) -> dict:
    result = {
        'has_coi': False,
        'coi_type': None,
        'evidence': [],
    }
    
    # Direct co-authorship (1 hop)
    if graph.has_edge(reviewer_id, applicant_id):
        edge = graph[reviewer_id][applicant_id]
        if edge['latest_year'] >= CURRENT_YEAR - COI_HORIZON_YEARS:
            result['has_coi'] = True
            result['coi_type'] = 'direct_coauthor'
            result['evidence'] = edge['papers']
            return result
    
    # Shared collaborators (2 hops)
    reviewer_neighbors = set(graph.neighbors(reviewer_id))
    applicant_neighbors = set(graph.neighbors(applicant_id))
    shared = reviewer_neighbors & applicant_neighbors
    
    for shared_author in shared:
        r_edge = graph[reviewer_id][shared_author]
        a_edge = graph[shared_author][applicant_id]
        
        if (r_edge['latest_year'] >= CURRENT_YEAR - COI_INDIRECT_HORIZON and
                a_edge['latest_year'] >= CURRENT_YEAR - COI_INDIRECT_HORIZON):
            result['has_coi'] = True
            result['coi_type'] = 'shared_collaborator'
            result['evidence'] = {
                'shared_author': shared_author,
                'reviewer_papers': r_edge['papers'],
                'applicant_papers': a_edge['papers'],
            }
            return result
    
    return result
```

For a review panel of 10 reviewers against 1 applicant, this runs in ~12ms. For the full department (27 faculty), it takes under 500ms.

## The hard problem: entity resolution

The publication databases return author names, not stable identifiers. "R. Bollepalli" and "Ravi C. Bollepalli" and "Ravi Bollepalli" are the same person, but string matching fails.

I used a two-stage approach:

**Stage 1: Deterministic matching**

```python
def normalize_author_name(name: str) -> str:
    """Smith, John A. → john smith"""
    parts = name.lower().replace(',', '').split()
    # Remove middle initials and suffixes
    parts = [p for p in parts if len(p) > 1 or p.isalpha()]
    return ' '.join(sorted(parts))  # Sort for commutative matching

def find_deterministic_match(
    candidate: str,
    known_authors: dict[str, str]  # canonical_id -> normalized_name
) -> str | None:
    normalized = normalize_author_name(candidate)
    for canonical_id, known_norm in known_authors.items():
        if normalized == known_norm:
            return canonical_id
    return None
```

This handles 96.5% of cases — same name, different formatting.

**Stage 2: GenAI disambiguation for ambiguous cases**

The remaining 3.5% are genuinely ambiguous: two researchers named "J. Kim" in the same field. I pass these to GPT-4 with institutional affiliation context:

```python
def resolve_ambiguous_author(
    candidate_name: str,
    candidate_affiliation: str,
    known_candidates: list[dict],
) -> str | None:
    prompt = f"""Given an author and a list of candidate researchers, identify which candidate (if any) this author is.

Author: {candidate_name}
Affiliation: {candidate_affiliation}

Candidates:
{json.dumps(known_candidates, indent=2)}

Return the canonical_id of the matching candidate, or null if none match.
Respond with JSON only: {{"canonical_id": "..." | null, "confidence": "high" | "medium" | "low"}}"""

    response = openai.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        response_format={"type": "json_object"},
    )
    
    result = json.loads(response.choices[0].message.content)
    if result['confidence'] in ('high', 'medium'):
        return result['canonical_id']
    return None
```

Combined, this achieved **96.5% entity matching accuracy** across 23,048 records, improving data completeness from 85% to 97.9%.

The 98% precision figure comes from manual validation: I spot-checked 200 random COI flags against manually verified ground truth. The 2% errors were all false positives (flagging borderline cases) — never false negatives. For COI detection, false positives are much safer than false negatives.

## What surprised me

**Graph topology reveals structure.** Running PageRank on the collaboration graph showed that 3 faculty members were hubs connecting otherwise-disconnected research clusters. That's an insight the manual process would never surface.

**GenAI disambiguation is expensive.** GPT-4o-mini costs ~$0.0001 per disambiguation call, but with 23,048 authors and ~3.5% ambiguous, that's ~800 API calls at entity load time. Caching the results and only re-disambiguating new records keeps ongoing costs near zero.

**Recency weighting matters a lot.** A co-authorship from 2010 shouldn't trigger a 2026 COI flag. The `latest_year` edge attribute with a sliding window cutoff is simple but exactly right.
